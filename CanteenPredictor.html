Core Formula
Demand = BaseDemand(item) × StudentScale
        × W_day(day_of_week)     // 0.60 Sun → 1.20 Fri
        × W_meal(meal_time)     // 0.50 Snack → 1.00 Lunch
        × W_weather(condition)    // 0.85 Hot → 1.15 Rainy
        × W_event(special_event)    // 0.40 Holiday → 1.40 Festival
        × ItemBias(item, meal_time)  // e.g. Idly×1.8 at Breakfast
        × ε                    // noise ∈ [0.90, 1.10]

Limitations & Recommended Upgrades
⚠️ Current Limitations
  • Synthetic baseline data (no real POS DB)
  • Linear multiplicative model ignores complex interactions
  • Weather manually input, not live-fetched
  • No semester / academic calendar awareness
  • Single-canteen model; no multi-branch support
🚀 Recommended Upgrades
  • Integrate real POS transaction history via Supabase
  • Train XGBoost / LSTM on 6+ months of live data
  • Add OpenWeatherMap API auto-fetch
  • Include college timetable and calendar data
  • Multi-item batch predictions with priority queue
